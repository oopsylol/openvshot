# 国内视频模型评测流程

## 目标
通过标准化样片与量化评分，选择主用视频模型与备用模型。

## 候选模型收集
- 收集 3-5 个国内可用的视频生成模型（API 或平台）
- 记录价格、并发、时延、合规文档、服务稳定性

## 标准化评测样片
- 统一脚本: 3 个场景、8-12 镜头
- 统一角色: 2 名主角、1 名配角
- 统一风格: 写实都市、室内+室外混合
- 输出规格: 9:16，24fps，单镜头 3-6 秒

## 评分维度与权重
- 写实度 25%
- 角色一致性 25%
- 时序稳定 15%
- 成本 15%
- 时延 10%
- API 稳定性 5%
- 合规文档完整度 5%

## 评测步骤
1. 用同一镜头清单 JSON 生成所有候选模型的样片
2. 对每个模型独立打分，并记录失败镜头与原因
3. 汇总评分表并计算加权总分
4. 选总分第一为主用模型，第二为备用模型
5. 记录模型特有参数与注意事项，写入 SOP

## 输出物
- 评分表：`evaluation/score_sheet.csv`
- 评测结论：主用与备用模型名称、适配参数、已知问题清单

## 注意事项
- 对同一镜头需固定 seed 或等效参数，保证可比性
- 所有模型输出均保留原始文件以便复检
- 若出现严重不稳定或合规问题，直接淘汰
