# SCU-OS v3.1（优化版）

## 概览
SCU-OS v3.1 用工业化流程将“灵感 -> 剧本 -> 视觉资产 -> 分镜 -> 生成 -> 归档”串联起来。优化版聚焦三件事：
- 输入输出标准化，减少返工
- 关键门禁明确，避免“没分镜就开机”
- 生成流程可编排，便于自动化

## 核心法则（不可违反）
- 角色长相必须锁定，严禁换脸
- 描述物理事实(材质/光影)，而非形容词
- 无分镜，不开机
- 视频生成使用中文原生指令

## 总流程（8 个阶段）
1) 选题与世界观设定
输入: 主题、受众、平台要求
输出: Logline、世界观设定、三幕/四幕概述
门禁: 3-5 场景内可闭合冲突

2) 角色设定与一致性包
输入: 世界观、人物关系
输出: 角色卡、参考图包、风格一致性片段
门禁: 每个角色至少 3 张参考图，外观描述可复述

3) 剧情大纲与分场剧本
输入: Logline、角色卡
输出: 分场剧本、对白草稿
门禁: 每场可拆为 2-4 个镜头

4) 七维分镜
输入: 分场剧本
输出: 分镜工程表、镜头目标
门禁: 每镜头有清晰目标与时长

5) 镜头级提示词
输入: 分镜工程表、角色一致性片段、场景描述
输出: 镜头清单 JSON
门禁: JSON 字段完整，提示词不冲突

6) 视频生成与回炉
输入: 镜头清单 JSON
输出: 单镜头视频素材
门禁: 写实度、时序稳定与一致性通过

7) 配音与字幕
输入: 台词、角色声音设定
输出: 配音文件、字幕文件
门禁: 听感自然、错别字率可控

8) 剪辑交付与归档
输入: 镜头素材、配音、字幕
输出: 成片与完整归档
门禁: 规格达标、资产可回溯

## 模块化映射
- Module A 文学车间: 阶段 1-3
- Module B 视觉车间: 阶段 2
- Module C 导演车间: 阶段 4
- Module D 铸造车间: 阶段 5-7
- Module E 工业归档: 阶段 8

## 标准化数据结构
- 镜头清单 JSON 规范见 `schema/shot_list.schema.json`
- 建议所有镜头以同一结构输出，便于自动化与可视化管理

## 统一 API 接入
- 文本与图像生成优先通过 OpenRouter 统一调用
- 使用“模型中心”拉取并筛选模型列表
- 视频生成使用七牛 Kling（七牛 AI 大模型推理 API）
- 备用方案: MemeFast（聚合接口）与火山云（兼容接口）
 - 火山云推荐 Ark Runtime SDK（`volcengine-python-sdk[ark]`）

## 资产管理
- 角色与场景参考图建议统一进入资产库
- 通过 COS 或 base64 保证引用可追溯与一致
- 生成图像可直接关联到角色/场景（含新建）
 - 支持 SQLite 持久化保存流程与配置

## 自动化建议
- 角色卡、场景、分镜统一结构化
- 镜头清单作为核心驱动数据
- 生成与回炉通过 status 字段管理

## 评测与选型
- 统一脚本与镜头清单评测 3-5 个候选模型
- 加权评分决定主用与备用模型
- 评测流程见 `evaluation/model_evaluation.md`
